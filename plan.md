# 背单词模式升级计划 (Plan Mode Upgrade)

为了更好地支持“从零开始背单词”的同学，我们将新增“背单词模式（计划模式）”，与现有的“积分模式”并存。

## 1. 核心业务流程
- **模式选择**：主界面增加“积分模式”和“背单词模式”切换。
- **背单词模式流程**：
    1. **设定目标**：用户设定每天背诵的数量（如 50 个词），系统自动从单词库中提取下一组词。
    2. **初次测试**：本轮依次测试这 50 个词。
    3. **测试报告**：测试完成后显示清单，标注哪些词对了，哪些词错了。
    4. **针对性复习**：用户可以查看错题的音标、释义。
    5. **循环重测**：**只针对本次没掌握的词**进行重测，直到当天选定的单词全掌握为止。
    6. **打卡成功**：当天单词全掌握后，记录历史并获得额外积分奖励。

## 2. UI 界面调整
### 主界面 (Setup Screen)
- 增加模式切换器：`[ 积分制 ]` `[ 计划制 ]`。
- **计划模式專屬配置**：
    - 显示“当前进度”（例如：已背完 250/2000 词）。
    - 输入框：设置每日背词数量（默认 50）。
    - 按钮：`开始今日学习`。

### 游戏界面 (Game Screen)
- 顶部进度条：由“当前积分/目标积分”变为“当前词数/本组总词数”。
- 增加“掌握情况”实时显示。

### 评审/结果界面 (Review Screen) - **新增界面**
- 显示本轮测试的统计信息（正确率）。
- 单词分类列表：
    - ✅ **已掌握** (Correct)。
    - ❌ **未掌握** (Wrong) - 点击可展开看提示复习。
- 按钮：`针对错词进行下一轮重测` 或 `全部掌握，确定打卡`。

## 3. 数据结构更新 (`localStorage`)
在 `state` 中增加：
```javascript
{
    mode: 'point', // 'point' | 'plan'
    currentIndex: 0, // 当前在总词库 (words.js) 中的索引
    dailyGoal: 50,  // 每天默认计划数
    currentBatch: [ // 当前正在背的一组词
        { wordIndex: 123, status: 'wrong' }, // status: 'correct' | 'wrong' | 'untested'
        { wordIndex: 124, status: 'correct' }
    ]
}
```

## 5. 预期效果
- **积分模式**：保持不变，适合温故知新，查漏补缺。
- **计划模式**：每日定额，强迫复习（必须全对才能完成），适合系统性学习。
